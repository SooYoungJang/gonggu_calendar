## 1. UX 라이팅 원칙

### 1.1 톤 & 매너

| 원칙 | 설명 | 예시 (좋음) | 예시 (나쁨) |
|------|------|-------------|-------------|
| **간결함** | 핵심만 전달, 불필요한 수식어 제거 | "승인되었습니다" | "해당 제보가 성공적으로 승인 처리되었습니다" |
| **명확함** | 모호한 표현 금지, 구체적으로 | "URL 형식이 올바르지 않습니다" | "입력값에 문제가 있습니다" |
| **친근함** | ~합니다 체, 존댓말 유지, 따뜻한 느낌 | "확인 후 다시 시도해주세요" | "에러가 발생했습니다. 관리자에게 문의하세요" |
| **일관성** | 용어 통일 (제보, 검수, 승인, 반려 고정) | "제보를 검수합니다" | "신청을 리뷰합니다" / "문의를 체크합니다" |

### 1.2 금지 용어

| 금지 표현 | 대체 표현 | 이유 |
|-----------|-----------|------|
| "클릭하세요" | "선택하세요", "눌러주세요" | 터치/키보드 환경 고려 |
| "에러" | "오류" | 한글화 |
| "FAILED" | "실패" | 한글화 |
| "너" / "당신" | 생략 또는 "님" | 존중감 |
| "관리자에게 문의" | 구체적 액션 제시 | 무책임함 방지 |
| "알 수 없는 오류" | 진짜 원인 파악 후 명시 | 신뢰도 |

## 8. 접근성 & 다국어 고려

### 8.1 ARIA 레이블 (스크린 리더)

```tsx
// 버튼
<button aria-label="제보 승인">승인하기</button>

// 토스트
<div role="alert" aria-live="polite">
  ✅ 승인 완료되었습니다
</div>

// 에러 메시지
<p role="alert" aria-live="assertive">
  제목을 입력해주세요
</p>

// 로딩 상태
<div aria-busy="true" aria-label="데이터 로딩 중">
  <Skeleton />
</div>

// 빈 상태
<section aria-label="빈 결과">
  <p>아직 접수된 제보가 없습니다</p>
</section>
```

## 9. 일관성 체크리스트

- [x] 모든 버튼: "~하기" 체계 통일 (승인하기, 반려하기, 저장하기, 취소하기)
- [x] 모든 상태: "~됨" 체계 통일 (승인됨, 반려됨, 대기 중)
- [x] 모든 오류: 사용자 행동 유도 (무엇을 어떻게 해야 하는지)
- [x] 모든 빈 상태: 아이콘 + 메시지 + 액션 (선택)
- [x] 모든 확인: 되돌릴 수 없는 작업은 확인 한 번 더
- [x] 모든 알림: 적절한 aria-live 속성
- [x] 존댓말 일관: ~하세요, ~입니다, ~주세요
- [x] 용어 통일: 제보(고정), 검수(고정), 공구(고정)
- [x] 축약어 금지: "ㅇㅋ" "ㄱㅅ" 등 비공식 표현 사용 금지
