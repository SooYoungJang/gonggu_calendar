## Variant: Dense Utilitarian (Mobile Submit)

### Design stance
Superhuman/Linear 스타일의 고밀도 도구형 접근. 카드 래퍼 제거, 필드를 바로 나열, 스티키 푸터 액션, 압축된 타이포그래피(14px 베이스), IBM Plex Sans로 기술적/전문적 인상. 한 화면에 최대한 많은 필드를 보여주어 스크롤 최소화.

### Key choices
- **Layout**: 래퍼 없는 직접 필드 나열, 상단 필수/선택 섹션 헤더로 구분, 스티키 푸터 버튼
- **Typography**: IBM Plex Sans, 20px 헤드라인, 14px 베이스, 12px 라벨(대문자), 모노스페이스 가능
- **Color**: 파란색(#0066cc) 액센트, 연한 회색 표면, 최소한의 시각적 장식
- **Interaction**: 즉각적 인라인 검증, 압축된 토스트(하단), 키보드 중심 네비게이션, 버전 표시
- **Spacing**: 12px 기본, 16px 섹션, 8px 필드 간격 (가장 조밀)

### Trade-offs
- **Strong at**: 정보 밀도, 파워 유저 효율, 한 화면 가시성, 빠른 입력 완료, 기술적 브랜드 인상
- **Weak at**: 초보자 인지 부하, 시각적 여백 부족으로 답답할 수 있음, 터치 타겟 작음(모바일)

### Best for
- 반복 제보하는 파워 유저, 관리자/운영자용 빠른 입력, 데스크톱 웹 우선, 효율성 중시 컨텍스트
